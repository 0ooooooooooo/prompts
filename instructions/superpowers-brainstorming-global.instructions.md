# Global Superpowers Brainstorming Rule

- Before answering coding or implementation requests, invoke the superpowers `brainstorming` skill first.
- Follow the brainstorming flow to clarify intent before implementation or conclusions.
- If the request is ambiguous, ask one clarifying question at a time.
- For simple non-coding questions, respond directly without mandatory brainstorming.
- Do not skip brainstorming for coding requests unless the user explicitly asks to bypass it.
- Keep final answers aligned with the decisions made during brainstorming.
